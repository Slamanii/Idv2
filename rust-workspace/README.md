# idv2 — Rust Workspace

Four on-chain Pinocchio programs + the off-chain enclave, CLI, dashboard, and test harness. Ownership of each layer, on-chain account layouts, and the PKCS#11 contract live in `../docs/`.

## Layout

```
rust-workspace/
├── programs/                 Pinocchio programs (BPF)
│   ├── election_registry/    ElectionAccount, candidate roster, phase, agg pubkey
│   ├── voter_registry/       Commitment Merkle tree (depth 28) + nullifier PDAs
│   ├── ballot/               cast ix — hot path; receipt-free (returns Ok(()))
│   └── tally/                Append-only sharded counters
├── clients/                  idv2-admin + idv2-verify (see ../docs/CLI.md)
├── enclave/                  PKCS#11 wrapper (cryptoki), HKDF, counter, WOTS driver
├── dashboard/                Aggregation server + signed-tuple WebSocket broadcast
└── tests/                    solana-program-test harness
```

## Quickstart

From the project root (`../`), not from inside `rust-workspace/`:

```sh
./scripts/install.sh      # Rust + Solana CLI + SoftHSM2
./scripts/keygen.sh       # authority, relayer, aggregation, 4× program keypairs
./scripts/airdrop.sh      # 2 SOL each to authority + relayer (devnet)
./scripts/deploy.sh       # cargo build-sbf + solana program deploy × 4
```

Or, just to check everything compiles:

```sh
cd rust-workspace && cargo check --workspace
```

First build pulls a large dep graph (Solana SDK is heavy); expect 5–10 minutes.

## Program IDs

Program IDs are the pubkeys of the keypairs in `.keys/programs/` generated by `keygen.sh`. They're stable across redeploys because `solana program deploy --program-id <keypair>` uses the provided keypair rather than minting a new one.

See `../docs/SETUP.md` for the full bring-up narrative and `../docs/PROGRAMS.md` for account layouts.
